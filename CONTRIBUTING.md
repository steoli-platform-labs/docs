# Contributing

Thank you for contributing to the AWS Platform Engineering labs.

## Before Opening a Pull Request

- Keep changes scoped to one clear purpose.
- Follow the repository boundaries and naming conventions used in the labs.
- Update documentation when behavior changes.
- Run all relevant validation commands.
- Do not include credentials, tokens, state files or generated secrets.

## Documentation Changes

- Use English.
- Keep headings descriptive.
- Use fenced code blocks with a language identifier.
- Ensure commands can be copied without hidden placeholders.
- Mark user-supplied values using angle brackets, for example `<github-organization>`.

## Pull Requests

A pull request should include:

- What changed
- Why it changed
- How it was validated
- Any follow-up work or known limitation
